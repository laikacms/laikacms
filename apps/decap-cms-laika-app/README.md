# @laikacloud/decap-cms-laika-app

Laika's all-in-one Decap CMS shell.

Decap CMS **v4.beta** wrapped in a Vite + TanStack (Router + Query) SPA and deployed as a Cloudflare
Worker.

> We bootstrap the CMS ourselves — no dependency on `decap-cms-app` (the upstream "all batteries"
> bundle that auto-init's and registers every backend/widget/editor-component on import). Instead
> the app pulls `decap-cms-core` directly and composes the React tree in
> [`src/client/DecapApp.tsx`](src/client/DecapApp.tsx):
>
> ```tsx
> <DecapCmsProvider config={cmsConfig}>
>   <App />
> </DecapCmsProvider>;
> ```
>
> Only the things we actually use are registered: `decap-cms-backend-github`,
> `decap-cms-widget-string`, `decap-cms-widget-datetime`, the two custom rich-text widgets and the
> locales bundle. Add more `CMS.registerWidget(...)` calls if a future collection needs them.
>
> Decap v4.beta isn't on npm yet. The monorepo's
> [`pnpm-local-override.json`](../../pnpm-local-override.json) (consumed by
> [`.pnpmfile.cjs`](../../.pnpmfile.cjs)) rewrites every `decap-cms-*` name to the local checkout at
> `~/Code/Personal/decap-cms/packages/decap-cms-*` when `enabled: true`. Run `pnpm install:dev` from
> the workspace root to install with overrides on; otherwise pnpm resolves the published 3.x
> tarballs. Inspired by the `decap-cms-app` package in the upstream v4.beta tree, but slimmer and
> pre-wired with our two custom rich-text widgets:

- `decap-cms-widget-lexicaleditor` — shadcn-editor-derived Lexical widget
- `decap-cms-widget-portabletext-editor` — Sanity's `@portabletext/editor`

Both widgets share the same 68-strong `portable-text-to-*-mapper` catalogue (registered globally on
widget load), so an author can paste in any of markdown, HTML, JATS, Pandoc JSON, Confluence storage
format, Notion blocks, or 60+ others and the right mapper picks it up.

## Layout

```
apps/decap-cms-laika-app/
├── index.html
├── package.json
├── tsconfig.json
├── vite.config.ts
├── wrangler.toml
└── src/
    ├── client/
    │   ├── main.tsx              # SPA bootstrap (TanStack Router + Query)
    │   ├── cms.ts                # lazy Decap CMS import + widget registration
    │   ├── config.ts             # inline Decap config (collections, backend)
    │   ├── routeTree.gen.ts      # auto-generated by router plugin
    │   └── routes/
    │       ├── __root.tsx
    │       ├── index.tsx         # landing page
    │       └── admin.tsx         # mounts Decap CMS
    └── server/
        ├── index.ts              # Hono worker (auth + health + asset fallback)
        └── auth.ts               # GitHub OAuth proxy for Decap's backend
```

## Develop

```sh
pnpm --filter @laikacloud/decap-cms-laika-app dev
```

Vite serves the SPA on http://localhost:3200/. The `@cloudflare/vite-plugin` mounts the worker in
front of it so `/auth/*` and `/api/*` behave the same in dev as in production.

## Configure the backend

Point at your content repo by editing `src/client/config.ts`:

```ts
backend: {
  name: 'github',
  repo: 'your-org/your-content-repo',
  branch: 'main',
  base_url: '',
  auth_endpoint: 'auth',
},
```

Create a [GitHub OAuth app](https://github.com/settings/applications/new):

- Homepage URL: your worker's URL
- Authorization callback URL: `${origin}/auth/callback`

Then store the credentials as Wrangler secrets (they are read by the worker, never sent to the SPA):

```sh
wrangler secret put GITHUB_OAUTH_CLIENT_ID
wrangler secret put GITHUB_OAUTH_CLIENT_SECRET
```

## Deploy

```sh
pnpm --filter @laikacloud/decap-cms-laika-app deploy
```

Runs `vite build` (output `dist/client/`) then `wrangler deploy`.
