# starter-feathers-blog

Minimal blog built with [Feathers.js v5](https://feathersjs.com) and LaikaCMS. Demonstrates the
**API-first / headless CMS** pattern:

- **Feathers service** — `PostsService` wraps `laika.documents.*` behind a typed REST interface.
  Any Feathers transport (REST, Socket.io) can query CMS content.
- **`laika.documents.*` via `laikacms/compat`** — `runTask` / `collectStream` inside the service
  class, identical API to all other starters.
- **Client-side blog** — `public/index.html` fetches `/posts` JSON and renders without SSR.
- **Decap admin from CDN** — `decap-cms.js` loaded from unpkg; laika backend bundled by esbuild.

## Quick start

```bash
pnpm install
pnpm dev
```

Open <http://localhost:3000> for the blog and <http://localhost:3000/admin/> for the CMS editor
(dev auth — no login required). The REST API is at `http://localhost:3000/posts`.

## Feathers service pattern

```typescript
// src/services/posts.service.ts
import { collectStream, runTask } from 'laikacms/compat';
import { laika } from '../lib/laika.js';

class PostsService {
  // Feathers REST: GET /posts
  async find(_params?: Params): Promise<Post[]> {
    const { items } = await collectStream(
      laika.documents.listRecordSummaries({ folder: 'posts', type: 'published', ... }),
    );
    // Load full content per summary...
    return posts;
  }

  // Feathers REST: GET /posts/:id
  async get(id: Id, _params?: Params): Promise<Post> {
    const doc = await runTask(laika.documents.getDocument(`posts/${id}`));
    return { id, ...doc.content };
  }
}

// Register with Feathers
app.use('posts', new PostsService());
```

`GET /posts` → JSON array. `GET /posts/my-first-post` → JSON object.

## Doc gaps found

1. **`doc.content` is `Record<string, unknown>`** — you must cast or validate to get a typed `Post`.
   A `zodSchemaFromCollection()` helper (or TypeBox equivalent) would let you derive types directly
   from the Decap collection definition instead of duplicating field names.

2. **Two-step fetch pattern** — `listRecordSummaries` + `getDocument` per item works but is two
   round-trips per post. If LaikaCMS ever supports fetching full document content in a single
   streaming call (e.g. `depth: 2` in `listRecordSummaries`), the Feathers `find()` implementation
   becomes a one-liner.

## REST API

| Method | Path          | Feathers operation | Description            |
| ------ | ------------- | ------------------ | ---------------------- |
| GET    | `/posts`      | `find()`           | List all published posts |
| GET    | `/posts/:id`  | `get(id)`          | Fetch a single post    |

Feathers adds standard error handling: `404 NotFound` if a post doesn't exist.
