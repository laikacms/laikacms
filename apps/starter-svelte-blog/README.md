# starter-svelte-blog

Minimal blog built with **Svelte 5** (standalone SSR, no SvelteKit) and LaikaCMS. Shows what
SvelteKit abstracts away:

- **`createEmbeddedLaika`** — wires up filesystem storage, Decap config syncing, and the Decap
  JSON:API fetch handler.
- **`laika.documents.*` via `laikacms/compat`** — `runTask` / `collectStream` in Express route
  handlers, exactly like any other SSR framework.
- **Svelte 5 `render()` from `svelte/server`** — compiles `.svelte` components to HTML on the
  server. Returns `{ html, head }` — you splice into the template manually.
- **Vite `ssrLoadModule`** — in dev mode, Vite transforms `.svelte` files with the Svelte plugin.
  Without SvelteKit you call this yourself from an Express route handler.
- **Decap admin from CDN** — `decap-cms.js` from unpkg; laika backend bundled by esbuild.

## Quick start

```bash
pnpm install
pnpm dev
```

Open <http://localhost:3000> for the blog and <http://localhost:3000/admin/> for the CMS editor
(dev auth — no login required).

  admin/

## How Svelte SSR works without SvelteKit

```typescript
// In an Express route handler:
import type { ViteDevServer } from 'vite';

// Dev: Vite transforms .svelte → server JS via ssrLoadModule
const { renderBlogPage } = await vite.ssrLoadModule('/src/entry-server.ts');

// The entry calls svelte/server's render()
const { html, head } = renderBlogPage(posts);

// You must splice head + body into your template yourself:
const page = template
  .replace('<!--ssr-head-->', head)     // <svelte:head> content (title, meta tags)
  .replace('<!--ssr-body-->', html);    // rendered component HTML
```

**Why `laika.ts` lives in Express, not in `entry-server.ts`:** If you import the laika singleton
inside `entry-server.ts` (and thus inside `ssrLoadModule`), Vite re-evaluates it on every hot
reload, creating multiple `EmbeddedLaika` instances and filesystem watchers. Keep the singleton in
the Express layer and pass content as props to Svelte components.

## Doc gap found

`render()` from `svelte/server` returns `{ html, head }`. The `head` string contains any
`<svelte:head>` content (title, meta, link tags). Unlike SvelteKit, there is no automatic head
injection — you must splice it into your HTML template. This is worth calling out in the LaikaCMS
docs when describing Svelte integration patterns.

## How content reading works

```typescript
import { collectStream, runTask } from 'laikacms/compat';
import { laika } from './lib/laika.js';

// List published posts
const { items } = await collectStream(
  laika.documents.listRecordSummaries({
    pagination: { page: 1, perPage: 100 },
    folder: 'posts', depth: 1, type: 'published',
  }),
);

// Fetch a single post
const post = await runTask(laika.documents.getDocument('posts/my-first-post'));
```

## Auth modes

| Mode     | When to use                                    |
| -------- | ---------------------------------------------- |
| `dev`    | Local development — no credentials required    |
| `custom` | Production — provide `authenticateAccessToken` |

## Production build

```bash
pnpm build   # vite build (client) + vite build --ssr (server) + esbuild admin
pnpm start   # NODE_ENV=production tsx src/server.ts
```

`dist/` contains the production client assets; `dist/server/` contains the SSR bundle.
